Authentication — единый вход, JWT токен ▶
POST /login NEW Единый вход для SMC и RSE ▶
Новый API. Заменяет GET /conf-portal/tokens и GET /sys-portal/tokens. Возвращает JWT, который используется для всех запросов к SMC и RSE. Токен не инвалидируется при входе с другого устройства — параллельные сессии работают одновременно.

Единая точка входа в систему. После получения токена все последующие запросы передают его через заголовок Authorization: Bearer {token}.

Request body
{ "login": "api_operator", "password": "your_password" }

passwordstring*Пароль пользователя

Conference management — создание, управление, участники ▶
POST /conf-portal/conferences Создать конференцию ▶

Создаёт конференцию в SMC. При enableLiveBroadcast: true в ответе возвращается liveAddress. Поля guestPassword и chairmanPassword возвращаются в ответе — необходимо сохранить для веб-подключения участников.

Live streaming & Web join — прямая трансляция и браузерное подключение ▶
GET /live/conference/{conferenceId} NEW HLS поток активной конференции ▶
Новый API. Заменяет liveAddress из GET /conferences/record/{id} который возвращал ссылку на веб-страницу Huawei. Теперь возвращает чистый HLS поток без редиректа.

Возвращает HLS-плейлист (.m3u8) прямой трансляции активной конференции. Может быть передан напрямую в HTML5 video player или hls.js без дополнительной авторизации на клиентской стороне.

Path parameters
NameTypeRequiredDescription
conferenceIdstring (uuid)*ID активной конференции
Response 200 — application/vnd.apple.mpegurl
#EXTM3U #EXT-X-VERSION:3 #EXT-X-TARGETDURATION:4 #EXT-X-MEDIA-SEQUENCE:0 #EXTINF:4.000, output_0.ts #EXTINF:4.000, output_1.ts #EXTINF:4.000, output_2.ts
Responses
200 OK 404 Conference not active 401 Unauthorized
GET /live/record/{segmentName}.ts NEW Сегмент HLS потока ▶
Новый API. Авторизация по JWT в заголовке — вместо cookie RSE сессии которая привязана к домену RSE и недоступна с другого домена.

Возвращает бинарный TS-сегмент HLS потока. Вызывается автоматически плеером при воспроизведении .m3u8 плейлиста.

Path parameters
NameTypeRequiredDescription
segmentNamestring*Имя сегмента без расширения, напр. output_3
Responses
200 OK — video/mp2t (binary) 404 Segment not found
POST /conference/{conferenceId}/join NEW Параметры для браузерного подключения (WebRTC) ▶
Новый API. Заменяет GET /smc_public/DataConferenceJoin/DataConferenceJoin.html который возвращает HTML страницу Huawei и перенаправляет пользователя на чужой сайт.

Возвращает параметры для WebRTC подключения к конференции прямо из браузера. Участник передаёт видео и аудио со своего устройства в ВКС Huawei без установки приложений и без перехода на сайт Huawei. Для подключения используйте sdp_offer_endpoint и ice_servers из ответа.

WS wss://smc.sud.uz/webrtc/{conferenceId} NEW WebRTC signaling — SDP обмен ▶
Новый API. WebSocket endpoint для SDP обмена и ICE negotiation. После завершения signaling медиапоток идёт напрямую через DTLS/SRTP — минуя сервер.

Браузер устанавливает WebSocket соединение используя sdp_offer_endpoint и join_token из POST /conference/{id}/join, передаёт SDP offer и получает SDP answer от MCU Huawei.

Server → Client (SDP answer)

GET /rse/rest/bnms/download/limit/3rdParty/downloadMedia Скачать файл записи (.ts) ▶

Скачивает файл записи с RSE сервера. Требует ticket полученный из /recordAddress. Ticket действует 3 минуты — необходимо инициировать скачивание сразу после получения.

Query parameters
NameTypeRequiredDescription
confIdstring (uuid)*ID конференции
recordIdstring (uuid)*ID файла записи
mediaTypeenum*videoHD | videoSD | videoAux
ticketstring*Ticket из /recordAddress. TTL: 3 минуты
Responses
200 OK — video/mp2t (binary .ts) 400 Ticket expired or invalid 401 Unauthorized
